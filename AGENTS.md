# Repository Guidelines

## Project Structure & Module Organization
- `src/` contains the C++20 application code, grouped by subsystem (e.g., `gpu/`, `image/`, `window/`, `roll/`, `state/`, `ocio/`).
- `assets/` holds UI assets, OCIO configs, and fonts used at runtime.
- `conanfile.py` and `conan-recipes/` define third‑party dependencies (OpenColorIO patches live under `conan-recipes/opencolorio/patches/`).
- `Build-debug/` and `Build-release/` are generated by the build scripts and contain the compiled app (`bin/Filmvert` or `Filmvert.app` on macOS).
- `Research/` contains workflow reference artifacts (not built).

## Build, Test, and Development Commands
- `./build.sh Debug` or `./build.sh Release`: Linux build via Conan + CMake; outputs to `Build-*/bin/`.
- `./build_mac.sh Debug` or `./build_mac.sh Release`: macOS universal build (arm64 + x86_64), then stitches into `Build-*/Filmvert.app`.
- `conan profile new default --detect`: generate missing Conan profiles if the build warns.

## Coding Style & Naming Conventions
- Indentation: 4 spaces; braces generally on the same line as declarations.
- Naming: files are `camelCase` or `snake_case` by subsystem; types are `CamelCase`/`lowerCamelCase` (e.g., `mainWindow`, `ocioProcessor`).
- No explicit formatter config; keep style consistent with nearby code.

## Testing Guidelines
- There are no automated tests in this repository.
- Validate changes by running the app and exercising the affected UI or processing path.
- For fixes in image processing or color management, include a before/after note and sample asset details in the PR description.

## Commit & Pull Request Guidelines
- Commit history favors short, imperative summaries (e.g., “Fix Conan v2 generation”, “Bug fixes, UI Improvements”).
- Keep commits focused; include platform context when relevant (Windows/macOS/Linux).
- PRs should include: a concise description, reproduction/verification steps, and screenshots or GIFs for UI changes.
- For bug reports or fixes, include log file paths when relevant:
  - macOS: `/Users/Shared/Filmvert/Filmvert.log`
  - Windows: `C:/ProgramData/Filmvert/Filmvert.log`
  - Linux: `~/.local/Filmvert/Filmvert.log`

## Configuration & Dependencies
- Dependencies are managed with Conan; update `conanfile.py` and any custom recipes together.
- OCIO configs are stored under `assets/` and are expected at runtime; update these with care.
